Server Preparation

Run the automated setup script to install dependencies and validate the server.

ℹ

The getstarted.sh script will install Docker, Nginx, validate mount points, check ports, and create required directories.

Step 1: Download the setup script

bash 
wget https://raw.githubusercontent.com/NCG-Africa/EdgeTelemetryDeployment/master/getstarted.sh

Step 2: Make it executable

bash 
chmod +x getstarted.sh

Step 3: Run the script with sudo

bash 
sudo bash getstarted.sh
⚠

Interactive Prompts

The script will ask for PostgreSQL host and port. Use the values you configured in Step 2.

What the script does:

  • ✓ Validates OS (Ubuntu 24.04 LTS), RAM (4GB+), and swap (16GB recommended)
  • ✓ Checks mount points: /, /var (100GB), /opt (80GB), /data (200GB)
  • ✓ Installs Docker Engine, Docker Compose, Nginx, make, unzip, curl
  • ✓ Validates port availability (80, 443, 8001-8003, 8080, 6379)
  • ✓ Tests PostgreSQL connectivity
  • ✓ Creates directories: /opt/edgetelemetry, /opt/edge-telemetry, /data/edgetelemetry
✓

Expected Completion Time

5-10 minutes depending on internet speed and system updates.

Common Issues:

  • Permission denied: Ensure you're running with sudo
  • Mount warnings: Review output and provision dedicated volumes for production
  • Port conflicts: Stop conflicting services or adjust port mappings
  • PostgreSQL unreachable: Verify DB host/port and firewall whitelist
  • Docker group: Run newgrp docker after script completes

Deploy Services

Start all EdgeTelemetry services using Docker Compose.

⚠

Make sure your .env file is in the project root before deploying.

bash

bash 
make prod-up
ℹ

This will start all services in detached mode. Initial startup takes 2-3 minutes.

Common Issues:

  • Docker not installed: Run sudo apt install docker.io docker-compose -y
  • Permission denied: Add user to docker group sudo usermod -aG docker $USER
  • Port conflicts: Check if ports are already in use

Configure Nginx & SSL

Set up reverse proxy and optional SSL certificates for production.

ℹ

SSL is optional for POC but required for production deployments. Configure your domain name and SSL certificates here.

Leave empty to use IP address only (POC mode).

Step 1: Create Nginx configuration

nginx.conf 
upstream telemetry_collector {
    server localhost:8001;
}

upstream telemetry_processor {
    server localhost:8002;
}

upstream dashboard_api {
    server localhost:8003;
}

upstream kafka_ui {
    server localhost:8080;
}

server {
    listen 80;
    server_name your-domain.com;

    location /collector/ {
        proxy_pass http://telemetry_collector/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /processor/ {
        proxy_pass http://telemetry_processor/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/ {
        proxy_pass http://dashboard_api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /kafka-ui/ {
        proxy_pass http://kafka_ui/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Step 2: Deploy Nginx configuration

bash 
sudo nano /etc/nginx/sites-available/edgetelemetry

Paste the configuration above, save (Ctrl+O), and exit (Ctrl+X).

Step 3: Enable the site

bash 
sudo ln -s /etc/nginx/sites-available/edgetelemetry /etc/nginx/sites-enabled/
bash 
sudo nginx -t
bash 
sudo systemctl reload nginx
⚠

SSL Configuration (Production Only)

For production deployments, SSL is mandatory. Follow the steps below to configure Let's Encrypt SSL.

Step 4: Install Certbot (SSL - Production Only)

bash 
sudo apt install certbot python3-certbot-nginx -y

Step 5: Obtain SSL certificate (Production Only)

bash 
sudo certbot --nginx -d your-domain.com

Certbot will automatically configure SSL and set up auto-renewal.

🔒

Prerequisites for SSL

  • Domain name must point to your server's IP address (A record)
  • Port 80 and 443 must be open in your firewall
  • Valid email address for Let's Encrypt notifications
✓

After SSL Configuration

Your services will be accessible via HTTPS at your domain name. Certbot will auto-renew certificates before expiry.

Common Issues:

  • Nginx test fails: Check syntax errors in configuration file
  • Port 80/443 conflicts: Stop other web servers using these ports
  • Certbot fails: Verify DNS A record points to server IP
  • Domain not resolving: Wait for DNS propagation (up to 48 hours)
